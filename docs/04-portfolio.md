# 04 — 유스케이스 발굴·우선순위·가치 가설

> [문서 03](03-alternative.md)의 루프에 "무엇을" 넣을지 정하는 문서입니다. 실제 작성은 [use-case-canvas 워크시트](../worksheet/use-case-canvas.md)로 이어집니다.

증분 AX는 좋은 후보를 고르는 데서 절반이 결정됩니다. 이 문서는 유스케이스를 발굴하는 법, 가치 가설로 정리하는 법, 우선순위를 매기는 법, 그리고 확률적 성과의 가치를 다루는 법을 제시합니다.

---

## 4.1 발굴 — 업무 페인에서 후보 찾기

기술 데모가 아니라 업무의 고통에서 출발합니다. 후보를 찾는 세 가지 경로입니다.

- **시간 인벤토리** — 부서별로 사람이 반복적으로 많은 시간을 쓰는 업무를 나열합니다. 검색·요약·작성·분류·확인 같은 패턴이 자주 보입니다.
- **현업 워크숍** — 실제 담당자에게 "가장 지겹고 반복적인 일"과 "전문성이 병목인 일"을 직접 받습니다.
- **품질·리스크 지점** — 실수·누락·지연이 자주 발생하는 지점을 찾습니다. AI 증강이 품질을 올릴 여지가 큽니다.

발굴 단계에서는 거르지 않고 폭넓게 모읍니다. 선별은 다음 단계에서 합니다.

## 4.2 가치 가설 캔버스

후보를 다음 항목으로 정리하면 검증 가능한 가설이 됩니다. 한 장으로 적습니다.

| 항목 | 내용 |
|------|------|
| 대상 사용자 | 누가 쓰는가 (구체적 부서·역할·규모) |
| 대상 업무 | 어떤 업무의 어떤 단계인가 |
| 현재 비용 | 지금 그 일에 드는 시간·비용·품질 손실 |
| AI 개입 지점 | 흐름의 어디에 AI가 끼어드는가 |
| 기대 효과 | 무엇이 얼마나 좋아질 것으로 믿는가 |
| 성공 지표 | 무엇을 보면 성공인가 (측정 가능하게) |
| 중단 기준 | 무엇을 보면 접는가 |
| 데이터 의존 | 어떤 사내 지식·데이터가 필요한가, 지금 상태는 |
| 주요 리스크 | 품질·규제·신뢰·보안 측면의 위험 |

이 캔버스가 [문서 03](03-alternative.md)의 파일럿 게이트로 그대로 이어집니다.

## 4.3 우선순위 — 가치와 실현가능성

후보를 두 축으로 배치합니다.

- **가치 축** — 효과의 크기와 전략적 중요도.
- **실현가능성 축** — 데이터 준비도, 기술 난이도, 리스크, 조직 수용성을 합친 것.

실현가능성에 **데이터 준비도**를 반드시 포함하는 것이 핵심입니다. [문서 01](01-diagnosis.md)의 데이터 부채 안티패턴은 가치만 보고 데이터 상태를 무시할 때 생깁니다.

| 구간 | 해석 | 행동 |
|------|------|------|
| 고가치·고실현 | 우선 파일럿 | 즉시 thin-slice 착수 |
| 고가치·저실현 | 전략 베팅 | 실현가능성 병목(주로 데이터)을 먼저 해소 |
| 저가치·고실현 | 빠른 효율화 | 여력이 있으면 가볍게 처리 |
| 저가치·저실현 | 보류 | 지금은 하지 않음 |

## 4.4 가치 유형

가치를 한 종류로만 보지 않습니다. 유형에 따라 측정 방식이 다릅니다.

- **효율화** — 같은 일을 더 적은 시간·비용으로. 측정이 비교적 쉬움.
- **품질 향상** — 오류·누락 감소, 일관성 향상. 품질 지표로 측정.
- **신규 역량** — 전에 못 하던 것을 가능하게. 단기 ROI보다 옵션 가치로 평가.
- **리스크 감축** — 규제 위반·사고 예방. 회피된 손실로 평가.

## 4.5 확률적 성과의 가치 평가

[문서 01](01-diagnosis.md)의 DX식 KPI 강제 안티패턴을 피하려면, 착수 전에 단일 ROI 숫자를 못 박는 대신 다음으로 다룹니다.

- **점추정 대신 범위** — "X% 절감"이 아니라 "낙관–비관 범위"로 제시하고, 파일럿으로 범위를 좁혀 갑니다.
- **기대값** — 성공 확률과 효과를 곱한 기대 효과로 후보를 비교합니다.
- **옵션 가치** — 지금은 작아도 성공 시 큰 확장으로 이어지는 후보의 가치를 인정합니다. 파일럿은 본질적으로 "작은 비용으로 큰 가능성을 사 두는 옵션"입니다.
- **학습 가치** — 결과와 별개로, 그 파일럿이 조직에 남기는 재사용 자산(평가셋·파이프라인)의 가치를 따로 셉니다.

이렇게 다루면 불확실하지만 가치 있는 시도가 결재 단계에서 배제되지 않습니다.

## 4.6 포트폴리오 균형

후보를 개별로 고르지 않고 포트폴리오로 관리합니다.

- **빠른 효율화(quick win)** — 동력과 신뢰를 만드는 단기 성과. 초기에 한두 개 확보합니다.
- **전략 베팅** — 효과는 크지만 시간이 걸리는 후보. 데이터·플랫폼 준비와 병행합니다.
- **역량 투자** — 직접 가치보다 재사용 자산을 남기는 것이 목적인 파일럿. 플라이휠을 돌립니다.

초기에는 quick win에 무게를 두어 신뢰를 쌓고, 역량이 축적되면 전략 베팅의 비중을 올립니다. 한쪽으로 쏠리면, quick win만 하면 큰 가치를 놓치고 전략 베팅만 하면 동력을 잃습니다.

선정이 끝났다면, 이제 누가 어떻게 이 루프를 굴릴지로 넘어갑니다.
