# 기업용 AX(AI Transformation) 방법론 가이드

DX를 흉내 낸 어설픈 AX는 효과 없이 예산만 태웁니다. 이 가이드는 그 실패의 구조를 **진단**하고, 빅뱅 전사 전환 대신 **증거 기반 증분 AX**라는 대안을 제시하며, 그 대안을 **Private AI(사내 AI)로 실제로 갖추는 전략**까지를 다루는 공개 실무 가이드입니다.

특정 벤더 제품을 전제하지 않는 벤더 중립적 방법론이며, Private AI 구현 단계에서만 한 가지 구체 경로로 PAIF/VCF 기반 시리즈를 참조로 연결합니다.

## 이 가이드의 입장

> **AX는 DX의 연장이 아니라 다른 종류의 변화이며, "한 번 끝내는 프로젝트"가 아니라 "증거를 쌓으며 역량을 키우는 학습 루프"로 다뤄야 합니다.**

DX의 대상은 결정론적(규칙이 분명한 업무의 디지털화)이지만, AX의 대상은 확률론적(맥락에 따라 출력이 달라지는 AI)입니다. 성격이 다른 대상에 같은 방법론을 적용하면 도구는 도입됐는데 가치는 나오지 않는 전형적 실패로 이어집니다. 그래서 이 가이드는 빅뱅이 아니라 증분, 확정이 아니라 학습, 도구가 아니라 역량을 일관되게 주장합니다.

가이드는 **진단 → 대안 → 구현**의 3부로 구성됩니다.

| 부 | 묻는 질문 |
|----|-----------|
| 진단 | 왜 지금의 AX는 실패하는가, 우리는 지금 어디에 있는가 |
| 대안 | 그러면 어떻게 다르게 할 것인가 |
| 구현 | 이 방식을 Private AI로 어떻게 갖추고, 무엇부터 실행하는가 |

## 주요 용어

| 용어 | 설명 |
|------|------|
| **AX** | AI Transformation — AI로 업무 판단·실행을 증강·위임하는 전환 |
| **Use-case-back** | 기술이 아니라 구체적 업무 페인에서 역산하는 원칙 |
| **Thin-slice** | 한 업무를 끝에서 끝까지 작게 관통하는 얇은 수직 단면 파일럿 |
| **게이트** | 파일럿을 사전 기준으로 확장/중단 판정하는 관문 |
| **Two-way-door** | 가역 결정은 빠르게, 비가역 결정은 증거 축적 후 |
| **역량 플라이휠** | 파일럿마다 재사용 자산을 쌓아 다음 시도를 싸게 만드는 선순환 |
| **Platform-pull** | 공통 수요가 드러날 때 플랫폼을 끌어오는 원칙 |
| **Private AI** | 데이터가 조직 경계를 벗어나지 않은 채 사내에서 운영하는 AI |

전체 용어는 [A1 용어집](appendix/A1-appendix.md)을 참조하세요.
