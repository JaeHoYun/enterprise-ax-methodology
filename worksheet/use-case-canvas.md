# 유스케이스 가치 가설 캔버스

> [문서 04](../docs/04-portfolio.md)의 가치 가설 캔버스를 한 유스케이스마다 한 장에 채웁니다. 후보별로 복사해 사용하세요.

- **누가** — 현업 프로덕트 오너와 기획이 주도하고, 데이터 의존·실현가능성 칸은 개발·인프라·데이터 담당과 함께 채웁니다.
- **언제** — 유스케이스 발굴·선별 단계에서 후보마다 한 장씩 작성합니다.
- **채운 뒤** — 우선순위에서 채택된 캔버스를 [파일럿 게이트](pilot-gate.md)로 이관합니다(어느 칸이 어디로 가는지는 맨 아래 §7 이관 표 참고).

유스케이스 이름: ______________   작성일: ______________   현업 오너: ______________

## 1. 진단 연계

[성숙도 자가진단](maturity-self-assessment.md)에서 나온 병목과 이 후보를 잇습니다. 진단 없이 떠오른 아이디어면 그대로 두되, 가급적 병목 해소와 연결해 "왜 이 후보인가"를 근거로 남깁니다.

- 이 후보가 해소하는 병목 차원: ______________ (예: 데이터·지식 / 조직·인재)
- 연결 강도: ☐ 병목 직접 해소  ☐ 부분 기여  ☐ 무관(빠른 효율화 목적)

## 2. 가치 가설

| 항목 | 작성 가이드 | 내용 |
|------|-------------|------|
| 대상 사용자 | 부서·역할·규모를 구체적으로 |  |
| 대상 업무 | 업무의 어느 단계인가 |  |
| 현재 비용 | 측정 가능하게 — 예: 인당 주 N시간 × 인원, 재작업·오류 비율, 처리 지연 |  |
| AI 개입 지점 | 업무 흐름의 어디에 끼어드나 |  |
| 기대 효과 (가설) | 무엇이 좋아질 것이라 믿나 (방향) |  |
| 성공 지표 (측정) | 그 효과를 재는 수치·기준 — 파일럿 게이트의 성공 기준이 되는 토대 |  |
| 중단 기준 | 이 선을 못 넘으면 접는다 |  |
| 데이터 의존 | 필요한 사내 지식·데이터와 현재 상태(품질·접근·권한) |  |
| 주요 리스크 | 품질·규제·신뢰·보안 + 예상 위험 등급(저/중/고) |  |

> 기대 효과(가설)와 성공 지표(측정)는 다릅니다. 기대 효과는 "무엇이 좋아지나"의 방향이고, 성공 지표는 그것을 숫자로 재는 기준입니다. 성공 지표가 비면 파일럿에서 성공/실패를 판정할 수 없습니다.

## 3. 우선순위 배치

[문서 04](../docs/04-portfolio.md)의 가치 × 실현가능성 매트릭스에 배치합니다.

- 가치 (상/중/하): ______   근거: ______________
- 실현가능성 (상/중/하): ______   근거(데이터 준비도·기술 난이도·리스크·수용성): ______________
- 구간 판정: ☐ 우선 파일럿  ☐ 전략 베팅(실현가능성 병목 먼저 해소)  ☐ 빠른 효율화  ☐ 보류

## 4. 가치 유형

가치 유형은 성공 지표를 어떻게 잡을지를 결정합니다(효율화는 시간·비용, 품질은 오류율, 신규 역량은 옵션 가치). §2의 성공 지표와 일관되게 고릅니다.

- ☐ 효율화  ☐ 품질 향상  ☐ 신규 역량  ☐ 리스크 감축

## 5. 데이터 출발점

[문서 07](../docs/07-private-ai-strategy.md)의 출발점 판단입니다. 이 칸은 파일럿 게이트의 "실행 환경"으로 그대로 이관됩니다.

- 민감 데이터 포함 여부: ☐ 없음  ☐ 일부(익명화·합성 가능)  ☐ 규제·기밀로 외부 전송 불가
- 권장 출발점: ☐ 외부 API·샌드박스  ☐ 익명화 데이터로 외부 검증  ☐ 처음부터 사내(Private)

## 6. 남길 재사용 자산

이 파일럿이 [문서 03](../docs/03-alternative.md)의 플라이휠에 남길 자산을 미리 정합니다.

- ☐ 평가셋  ☐ 데이터·검색 파이프라인  ☐ 가드레일·프롬프트 패턴  ☐ 플랫폼 구성요소
- 메모: ______________________________________________

## 7. 채택 시 파일럿 게이트로 이관

구간 판정(§3)이 "우선 파일럿"이면 아래 매핑대로 [파일럿 게이트](pilot-gate.md)에 옮겨 적습니다. 캔버스의 풍부한 항목이 게이트의 판정용 항목으로 압축되는 지점입니다.

| 이 캔버스의 칸 | → 파일럿 게이트의 칸 |
|----------------|----------------------|
| 대상 업무 + AI 개입 지점 | 가치 가설(한 문장) |
| 성공 지표 | 성공(확장) 기준 |
| 중단 기준 | 중단(kill) 기준 |
| 주요 리스크 + 위험 등급 | 위험 등급별 필요 통제 |
| 데이터 출발점(§5) | 실행 환경 |
| 남길 재사용 자산(§6) | 남기기로 한 재사용 자산 |
