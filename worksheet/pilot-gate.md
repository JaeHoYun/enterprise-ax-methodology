# 파일럿 게이트 기록지

> [문서 03](../docs/03-alternative.md)의 파일럿 게이트(확장/중단 판정)를 기록합니다. 파일럿마다 한 장에 작성하고, 착수 시점과 판정 시점에 나눠 채웁니다.

- **누가** — 현업 오너가 주도하고, 실행은 개발·인프라, 위험 등급·통제는 거버넌스, 확장/중단 판정은 의사결정자가 맡습니다.
- **언제** — 파일럿 착수 시(가설·기준) 1차, 판정 시점(결과·판정) 2차로 나눠 채웁니다.
- **채운 뒤** — 확장이면 운영·다음 사이클로, 중단이면 학습 기록 후 재검증 또는 다음 후보로 넘깁니다. 게이트 기록은 결재 근거·감사 추적으로 보관합니다.

유스케이스 이름: ______________   현업 오너: ______________   위험 등급: ☐ 저 ☐ 중 ☐ 고
출처 캔버스: ______________ (작성일/버전: __________) (→ [유스케이스 캔버스](use-case-canvas.md) §7에서 이관)

## 1. 착수 시점에 적는다

대부분 [유스케이스 캔버스](use-case-canvas.md)의 §7 이관 표에서 옮겨 옵니다.

| 항목 | 작성 가이드 | 내용 |
|------|-------------|------|
| 가치 가설 (한 문장) | 캔버스의 대상 업무 + AI 개입 지점을 한 문장으로 |  |
| Thin-slice 범위 | 사용자·데이터·흐름을 어디까지 좁혔나 |  |
| 실행 환경 | 캔버스 데이터 출발점 — 외부 API / 익명화 외부 / 사내(Private) |  |
| 성공(확장) 기준 | 캔버스 성공 지표 기반, 측정 가능한 임계로 |  |
| 중단(kill) 기준 | 측정 가능한 임계로 |  |
| 비용 상한 | 매몰되기 전에 멈출 예산 |  |
| 기간 상한 (타임박스) | 길어지면 자동으로 게이트를 연다 |  |
| 위험 등급별 필요 통제 | 캔버스 주요 리스크를 등급화한 통제 |  |
| 남기기로 한 재사용 자산 | 캔버스 §6 |  |

> 착수 후에는 위 기준을 바꾸지 않습니다. 사후에 기준을 옮기면 게이트가 무력화됩니다.

## 2. 판정 시점에 적는다

| 항목 | 작성 가이드 | 내용 |
|------|-------------|------|
| 실제 결과 (성공 지표 측정값) | §1 성공/중단 기준과 같은 단위로 |  |
| 비용·기간 실적 (상한 대비) | §1 상한 대비 초과 여부 |  |
| 거버넌스 통제 충족 여부 | §1 필요 통제가 복수면 항목별 충족/미충족을 구분 |  |
| 실제로 남긴 재사용 자산 | §1 계획 자산별 산출/미산출 사유 + 등록 완료 여부(§3 등록처) |  |

- ☐ **확장(scale)** — 의사결정자: ______   다음 단계: ______________

**재사용 자산 등록처:** ______________ — 남긴 자산(평가셋·파이프라인·가드레일 등)을 다음 파일럿이 찾을 수 있는 곳에 등록합니다. 플라이휠을 닫는 칸입니다.
