# 투자 케이스 — 가치를 비용에 잇기

> [문서 04 §4.8](../docs/04-portfolio.md)의 투자 케이스를 한 장에 채웁니다. 채택된 유스케이스(또는 묶음)마다 한 장씩 작성합니다. 비용 산식은 여기서 다시 만들지 않고 [⑥ 07 TCO 모델](https://github.com/JaeHoYun/vcf-private-ai-sizing-cost/blob/main/docs/07-tco-cost-model.md)에 위임합니다.

- **누가** — 현업 오너·재무·플랫폼/인프라가 함께 채웁니다. 편익은 현업, 비용은 인프라·재무, 판정은 셋이 합의합니다.
- **언제** — 우선순위에서 채택된 후보를 결재·예산에 올리기 직전. 파일럿 실측이 나오면 갱신합니다.
- **채운 뒤** — 손익분기·민감도·세 안 비교로 투자 판단을 내리고, "회수 안 되는 조건"을 모니터링 항목으로 남깁니다.

대상 유스케이스(또는 묶음): ______________   작성일: ______________   분석 기간: ☐ 3년 ☐ 5년

> **이 워크시트는 단일 ROI% 칸을 두지 않습니다.** 착수 전 단일 숫자 못 박기는 04 §4.6의 안티패턴입니다. 대신 범위·손익분기·민감도로 다룹니다.

## 1. 편익 환산 — 세 바구니

세 바구니를 **하나의 금액으로 합치지 마십시오.** 효율화만 금액으로, 나머지는 별도 표기로 둡니다(04 §4.8.2).

### 1-A. 효율화 (노동시간 회수) — 금액 환산

`연간 회수가치 ≈ Σ_과업 (연간 처리 건수 × 건당 절감 시간) × 로딩 인건비 단가 × 채택률 × 품질 통과율`

|------|---------------|----------------|---------------------|--------|-------------|-------------|
|  |  |  | (견적/HR 입력) |  |  | (계산) |
|  |  |  |  |  |  |  |
