# 자산 처분 결정 매트릭스 (6R)

이 문서는 이미 운영 중인 AI 자산을 자산 단위로 검토하여, 무엇을 남기고(잔류) 무엇을 옮기고 회수하고 통합하고 재구성하거나 폐기할지를 근거에 묶어 결정하기 위한 채워넣기 양식입니다. 6R은 자산을 처분(disposition)하는 여섯 가지 행동 선택지를 가리키며, 클라우드 마이그레이션 분야에서 쓰이는 6R 분류를 AI 인프라 자산에 맞게 변형한 것입니다.

## 머리말 — 누가, 언제, 채운 뒤 어디에 쓰는가

- **누가**: 인프라 운영 책임자와 재무·보안 검토자가 공동으로 채웁니다. 각 자산의 단가·가동률은 운영 책임자가, 데이터 민감도·주권은 보안·법무 검토자가 출처를 댑니다.
- **언제**: 자산 인벤토리([문서 09](../docs/09-brownfield-assessment.md))를 1차 완료한 직후 채웁니다. 인벤토리가 비어 있으면 이 매트릭스는 근거 없는 추정이 되므로 선행 입력을 먼저 확보합니다.
- **채운 뒤**: 자산별 6R 결정과 그 근거는 진단 보고서의 S3(개선안)·S4(전환 프로젝트 착수) 절의 직접 입력이 됩니다. 정량 수치 자체는 이 양식에서 계산하지 않고, 외부 사이징·비용 시리즈 워크북의 산출값을 옮겨 적습니다.

### 방향 — 역방향(기존 자산 감사·정리)

이 양식은 이미 보유한 AI 자산을 감사하여 정리하는 역방향 동선입니다. 신규 운영 확장을 준비하는 전방 동선의 [scale-readiness 워크시트](scale-readiness.md)와 방향이 반대이므로, 두 양식을 혼용하지 않습니다.

### 빅뱅 회귀 경계

이 매트릭스는 "전사 청사진을 먼저 그린 뒤 한꺼번에 갈아엎으라"는 양식이 아닙니다. 회수·통합·폐기는 인벤토리 실측과 검증된 수요가 당길 때(pull) 자산 단위로 진화적으로 진행하는 결정이며, 이 점이 어긋나면 빅뱅 안티패턴([문서 01](../docs/01-diagnosis.md))으로 되돌아갑니다.

## 1. 처분 6R 정의 (AI 자산용 변형)

| 코드 | 행동 | AI 자산 맥락의 의미 |
|---|---|---|
| Retain | 잔류 | 베어메탈 GPU 등 현 위치·현 형태를 그대로 유지. 손대지 않는 것이 합리적인 자산 |
| Relocate | 이전 | 형태는 유지하되 위치만 옮김(예: 리전 이동, 코로케이션 이전) |
| Repatriate | 회수 | 퍼블릭 클라우드에서 프라이빗·온프렘으로 되가져옴 |
| Consolidate | 통합 | 중복·분산된 자산을 하나로 합침(중복 AI팀·중복 추론 엔드포인트 등) |
| Refactor | 재구성 | 형태·아키텍처를 바꿔 다시 구성(예: 공유 스케줄러 도입, 모델 서빙 계층 교체) |
| Retire | 폐기 | 더 이상 수요가 없거나 가치가 비용을 밑도는 자산을 종료 |

용어 풀이: **잔류**(Retain)는 "옮기지 않는다"는 적극적 결정이지 "아직 손대지 못했다"는 방치가 아닙니다. 두 상태를 구분해 기록합니다.

## 2. 자산별 6R 선택 표

한 행이 한 자산입니다. 자산 ID와 명칭은 인벤토리([문서 09](../docs/09-brownfield-assessment.md))의 식별자를 그대로 옮겨 적어 추적이 끊기지 않게 합니다. 6R 열에는 최종 선택 하나만 표기하고, 후보가 둘 이상이면 비고에 차순위와 보류 사유를 적습니다.

| 자산 ID | 자산 명칭·유형 | 현재 위치·형태 | 6R 선택 | 핵심 근거(요약) | 결정 신뢰도 | 비고(차순위·전제) |
|---|---|---|---|---|---|---|
| (인벤토리 ID) | (예: 추론 엔드포인트 A) |  |  |  | 상/중/하 |  |
|  |  |  |  |  |  |  |
|  |  |  |  |  |  |  |

위 표의 빈칸은 채우라고 비워둔 것입니다. 자산 수만큼 행을 늘립니다.

- **6R 선택** [필수]: 위 6코드 중 하나. 출처는 아래 3절 스코어링 결과와 4절 흐름.
- **핵심 근거** [필수]: 어떤 점수·판정이 이 선택을 끌어냈는지 한두 줄. 측정 방법은 해당 점수 항목 출처를 가리킴.
- **결정 신뢰도** [권장]: 입력 데이터의 충분성에 대한 자기 평가(상/중/하). '하'인 행은 S3 보고서에서 추가 실측 과제로 분리.
- **비고** [선택]: 차순위 R, 결정의 전제 조건, 재검토 시점.

## 3. 스코어링 가이드

각 자산을 다섯 축으로 채점합니다. 점수는 1~5 정수이며, 자동 계산은 없습니다. 정량 축(단가 손익분기·재작성·이전 비용)의 원수치는 외부 사이징·비용 워크북에서 옮겨 적고, 이 표에는 그 수치를 1~5로 환산한 등급만 기록합니다.

| 스코어링 축 | 1점(이전·정리 압력 낮음) | 5점(이전·정리 압력 높음) | 출처·측정 방법 |
|---|---|---|---|
| 데이터 민감도·주권 | 공개·비민감, 위치 제약 없음 | 규제·주권으로 국내·온프렘 강제 | 데이터 분류 정책, 법무·보안 검토 |
| 단가 손익분기 | 클라우드가 단가상 유리 | 자가 보유가 단가상 유리 | 비용 워크북의 GPU 시간당 단가 대비 |
| 재작성·이전 비용 | 이전이 거의 무비용 | 재작성·이전 비용이 큼 | 사이징·이전 견적 워크북 |
| 벤더 종속도 | 표준·이식 가능 | 특정 벤더 API·서비스에 강결합 | 아키텍처 실사, 종속 서비스 목록 |
| 사용 패턴 안정성 | 변동·예측 불가 | 예측 가능·지속 부하 | 12~24개월 사용량 로그 |

채점값의 다음 활용: 다섯 축 점수는 합산해 단일 순위로 쓰지 않습니다. 축마다 다른 R로 끌기 때문입니다(예: 데이터 주권 5점은 회수·잔류로, 벤더 종속도 5점은 재구성으로). 점수는 4절 흐름의 분기 입력으로만 씁니다.

### 점수에서 R로 가는 경향 (단정 아닌 참고)

- 데이터 민감도·주권 높음 → 회수·잔류 쪽으로 당김.
- 단가 손익분기에서 자가 보유 유리 + 사용 패턴 안정 → 회수 검토.
- 재작성·이전 비용 큼 → 잔류·재구성 쪽으로, 이전·회수는 신중.
- 벤더 종속도 높음 → 재구성으로 종속 완화 검토.
- 사용 패턴 안정성 낮음(변동적) → 클라우드 잔류 쪽으로.

## 4. 의사결정 흐름

자산마다 아래 순서로 점검합니다. 각 분기는 단정이 아니라 판단축이며, "예"가 곧 자동 결정은 아니고 다음 절의 회수 판정 기준과 교차 확인합니다.

1. 더 이상 쓰이지 않거나 가치가 운영비를 밑도는가? → 예이면 **폐기(Retire)** 후보.
2. 동일·유사 기능 자산이 중복되는가(중복 추론 엔드포인트·중복 AI팀 자산)? → 예이면 **통합(Consolidate)** 후보.
3. 데이터 주권·지연·데이터 중력이 현 위치를 강제하는가? → 예이면 현 위치 유지 방향(**잔류** 또는 **회수**).
4. 퍼블릭 클라우드에 있고 5절 회수 판정에 부합하는가? → 예이면 **회수(Repatriate)** 후보.
5. 형태는 유지하되 위치만 바꾸면 이득인가(리전·코로케이션)? → 예이면 **이전(Relocate)** 후보.
6. 벤더 종속·구조 비효율이 큰가? → 예이면 **재구성(Refactor)** 후보.
7. 위 어디에도 강하게 걸리지 않는가? → **잔류(Retain)** (방치가 아닌 적극적 유지 결정).

여기서 "데이터 중력"(data gravity)은 데이터가 쌓일수록 그 데이터를 옮기기 어려워지고 연산이 데이터 쪽으로 끌려오는 현상을 가리킵니다.

## 5. 회수(Repatriation) 판정 기준

아래는 회수 여부를 가르는 판단축입니다. 수치는 예시이며 자사 실측으로 갈음합니다. 단정형 기준이 아니라, 여러 축을 함께 보고 회수가 합리적인지 판단하는 잣대로 씁니다.

- **지속 가동률 대 손익분기**: 예측 가능한 지속 추론 워크로드는 지속 가동률이 손익분기(개념적으로 60~70% 부근)를 넘으면 온프렘이 유리해질 수 있습니다.
- **회수기간(payback)**: 동일 GPU의 클라우드 연환산 비용을 구매가와 견주어 회수기간을 따집니다.
- **사용 패턴 성숙도**: 클라우드에서 12~24개월 운영되어 사용 패턴이 예측 가능해졌는지 봅니다.
- **하이브리드 분할**: 변동적 훈련은 클라우드에 유지하고, 지속적 추론은 온프렘으로 가르는 분할이 더 합리적일 수 있습니다.
- **데이터 중력·지연·주권**: 이 셋 중 하나라도 회수를 강제하면 단가와 무관하게 회수가 당겨질 수 있습니다.

이 판정의 상세 근거와 회수 후 구체 구현(마이그레이션·통합 실행)은 [문서 07](../docs/07-private-ai-strategy.md)을 참조하며, 구현 단계는 벤더 중립 진단의 범위를 벗어나 외부 VCF Private AI 시리즈로 위임합니다.

## 6. 채운 뒤 연계

- 자산별 6R 결정과 근거는 보고서 **S3**(종합 개선안)의 자산 정리 계획 표로 옮깁니다.
- 회수·통합·재구성으로 분류된 자산 중 검증된 수요가 받쳐주는 항목은 보고서 **S4**(전환 프로젝트 착수)의 초기 대상 목록이 됩니다.
- 결정 신뢰도 '하'인 행은 S3에서 별도의 추가 실측 과제로 분리해, 근거가 갖춰진 뒤 재판정합니다.
